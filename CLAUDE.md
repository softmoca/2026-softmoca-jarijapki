# CLAUDE.md — 작업 프로토콜 (루트)

> 이 파일은 Claude Code가 이 저장소에서 작업할 때 **가장 먼저 읽는** 운영 지침이다.
> 상세 내용은 `/docs`의 각 문서를 참조한다. 이 파일은 "규율의 요약본"이다.

---

## 0. 한 줄 정의

첫 자취생이 **매번 흩어진 정보를 검색해 조립하던 시간**을 없애주는 자취 정착 도우미.
이번 스프린트 구현 범위 = **① 방 구할 때 체크리스트 + ② 자취 백과(큐레이션 지도)**.
청소 기능은 **비전에는 있으나 이번 스프린트에는 구현하지 않는다**(`docs/05-out-of-scope.md`).

---

## 1. 절대 규칙 (보안 · 네이밍)

1. 패키지명 · 도메인 · 서비스 문자열에 `woowa`, `woowahan`, `techcourse` 를 **절대 포함하지 않는다.**
2. 서비스명은 **`SERVICE_NAME` 상수 하나**로만 관리한다. (현재 placeholder: `자리잡기`)
   - UI·메타태그·문서 어디에도 서비스명을 하드코딩하지 않는다. 항상 상수를 참조한다.
   - 이름 변경 = 상수 한 줄 수정으로 끝나야 한다.
3. 저장소는 `woowacourse-personal`에 `2026-{닉네임}-{프로젝트명}` 규칙으로 생성한다.
4. repository > Settings > Security 에서 Secret scanning 을 켠다.

## 3. 이번 스프린트의 두 기능 — "만들기는 같이, 검증은 따로"

- **한 저장소 안에 체크리스트 + 백과를 함께 구현한다.** (같은 큐레이션 엔진을 공유하므로 자연스럽다)
- **그러나 인터뷰에서는 둘을 분리해서 검증한다.** 위험 가정이 서로 다르기 때문이다.
  - 체크리스트 위험 가정 = "초보는 **뭘 체크할지 자체를 모른다**" → 순서대로 던져주면 가치 있나
  - 백과 위험 가정 = **습관·유입** → "그 순간 반사적으로 유튜브 치는 대신 우리를 여나"
- 자세한 검증 프로토콜은 `01-validation-map.md`.

---

## 4. Claude Code 작업 시 지켜야 할 것

- 작업 시작 전 관련 `docs/*.md`를 먼저 읽는다. 스펙에 없는 화면·기능을 임의로 만들지 않는다.
- 백엔드·DB·로그인·결제 없음. **모든 데이터는 하드코딩 더미(`03-data.md`)**.
- 물리 작업(에어컨 청소, 실리콘, 타일 등)은 텍스트로 정면 설명하지 않는다. **검증된 외부 영상/자료로 링크아웃**한다. (백과 = 지도이지 콘텐츠 공장이 아니다)
- 화면을 늘리고 싶을 때: 먼저 `05-out-of-scope.md`의 자기진단 체크리스트를 통과하는지 확인한다.
- 커밋은 작게, 마일스톤 단위(`04-tech.md`)로.
